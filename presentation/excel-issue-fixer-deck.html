Excel Issue Fixer

Turn an Excel UAT / issue sheet into a planned, tested fix — classify every issue, propose Before/After edits, then implement with tests.

Presented by Krirerk Chootiang

01 / What it does

Sheet in, shipped fixes out.

Input

An .xlsx + your repos

An issue / UAT sheet plus your frontend and backend projects.

Output

3-pillar plan + code

All-Tasks board, Frontend plan, Backend plan — and the implemented fixes.

Guarantee

Approval before edits

Every fix is shown as Before/After and gated on your sign-off.

Quality

Tests every fix

Tests follow each change; status stays synced across all three docs.

Steps 01–02 / Parse & Select

Read the sheet, choose the codebases.

01 · Parse

Extract every row

Reads the Excel and pulls each issue — id, title, description, priority.

02 · Select

Pick Frontend & Backend

A popup lists auto-discovered candidates: angular.json for FE, .sln/.csproj for BE. Picks become FERoot / BERoot; "None" skips that side.

gable-dotnetfix-skills06 / 12

Steps 03–04 / Classify & Detect

Sort issues, read the stack.

03 · Classify

Backend · Frontend · Fullstack

Each issue tagged by signal words — .NET API terms vs Angular UI terms.

04 · Detect

Framework & versions

Auto-detect inside FERoot/BERoot — never hardcoded. Probes package.json, angular.json, .csproj, global.json.

gable-dotnetfix-skills07 / 12

Step 05 / Plan

Three linked pillars, before any code.

Pillar 1

All-Tasks board

Every issue with triage + status, grouped by layer.

Pillar 2

Frontend plan

Before/After per FE issue, phased by risk.

Pillar 3

Backend plan

Before/After per BE issue, phased.

A fullstack issue appears once in All-Tasks and in both plans — each showing only its layer's half.

gable-dotnetfix-skills08 / 12

Step 06 / Suggest — approval gate

Show the fix. Wait for sign-off.

Present

Before / After

Per issue, grouped backend-first then frontend.

Before

Real code

Read verbatim from FERoot/BERoot with file:line citations.

After

Minimal edit

Marked with FIXED / BUG comments — nothing more.

No files are edited until you approve all, approve a subset, or request changes.
